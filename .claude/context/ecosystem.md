## Directory Conventions

```
src/
├── components/        # UI (ui/ for shadcn, domain/ for business logic)
├── hooks/             # React hooks (internal/ for data layer)
├── pages/             # Route page components
├── types/             # TypeScript interfaces
├── integrations/
│   └── supabase/      # client.ts + types.ts
├── lib/               # Utilities (cn, etc.)
├── contexts/          # React contexts
└── services/          # API service layers
```

## Coding Standards

- Components (.tsx) should be under 150 lines — split into smaller components
- Logic goes in hooks: `hooks/use{ComponentName}.ts`
- File names: `with-this-kind-of-case`
- Atomic design for UI components
- Feature-based organization for domain code
